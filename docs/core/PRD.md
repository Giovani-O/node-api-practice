## Functional Requirements
- [ ] User can create new transactions;
- [ ] User can obtain a summary of their account;
- [ ] User should be able to list all transactions;
- [ ] User should be abe to get informations from a single transaction;

## Business Rules
- [ ] A transaction can be of type credit, that will sum to the total value, or debit that will subtract;
- [ ] It should be possible to identify a user between requests;
- [ ] Users should only be able to view transactions created by themselves;
